## 一、插件概览

### 设计初衷

本插件的目标是将**阅读、查词、复习、使用**四个环节整合在一起，解决以下痛点：

- 查词和笔记分散在不同软件，数据割裂
- 单词笔记是静态的，写作时无法直接调用
- 缺乏丰富的学习统计，难以看到学习进度

灵感来源于 LingQ，但完全本地化运行，且额外解决了写作辅助的问题。

### 核心功能清单

|功能|说明|
|---|---|
|查词|划词调用有道词典，含柯林斯例句、近义词辨析、同根词|
|添加笔记|每个单词支持多条笔记和例句，保存至本地 IndexDB|
|解析阅读|文章每个单词变为可点击按钮，高亮已记录单词，支持音频同步|
|统计面板|显示近 7 天每日学词量和累计总量|
|自动补全|联动 Various Complements，写作时提示已记单词（支持英→中、中→英）|
|间隔复习|联动 Spaced Repetition，将生词导出为闪卡进行间隔复习|

## 二、查词功能

### 操作方式
在任意笔记中选中单词，右键 → 点击**「查找单词/短语」**，自动打开查词面板。

也可点击左侧工具栏的**词典图标**手动打开面板，并在顶部输入框手动输入查词。
### 面板内容

- 各词性的含义、单复数形式、分词形式
- 柯林斯词典大量例句
- 近义词辨析

---

## 三、添加笔记

### 打开新词面板

点击左侧工具栏的**眼镜图标**，打开新词面板。

在笔记中划词查词时，单词会自动填充到该面板，直接查看释义并记笔记。

若该单词已记录过，查词时会自动显示之前的笔记，可修改后重新提交。

### 笔记字段说明

| 字段       | 说明                           |
| -------- | ---------------------------- |
| **表达**   | 单词或短语（必填）                    |
| **含义**   | 简短定义，用于复习提示，不必写长             |
| **类别**   | 单词 / 短语（多个词组成的短语必须选「短语」）     |
| **学习状态** | 无视 / 新学 / 眼熟 / 掌握 / 会了，共 5 档 |
| **标签**   | 可输入或选择已有 tag，无需加 `#` 前缀      |
| **笔记**   | 支持多条长笔记                      |
| **例句**   | 支持多条，含源句（必填）、翻译（可选）、出处（可选）   |

> ⚠️ 提交后按钮一直转圈表示保存失败，可能是数据库不存在或其他 bug。

## 四、解析阅读（核心功能）

### 文章笔记格式

新建一篇笔记，按以下格式组织内容：

```
---
langr: xxxx
langr-audio: "音频链接URL"
langr-origin: 《文章名称》
---

^^^article

（此处粘贴英文文章正文）

^^^words

^^^notes
```

**标记说明：**

| 标记                           | 作用                |
| ---------------------------- | ----------------- |
| `langr: xxxx`                | 声明该笔记可被插件识别为阅读材料  |
| `langr-audio`                | 音频链接，配置后可边听边读     |
| `langr-origin`               | 文章出处，自动填充到例句的出处字段 |
| `^^^article` 到 `^^^words` 之间 | 被识别为文章正文内容        |

### 进入阅读视图

两种方式：

- 点击右上角「更多选项」→「**打开为阅读模式**」
- 直接点击左侧**书本形按钮**

### 颜色含义

|颜色|含义|
|---|---|
|淡蓝色|数据库中未记录的新词|
|橙黄色|已记录、正在学习中的单词|
|无背景|已标记为「无视」的熟词|

### 添加单词

点击任意单词 → 自动填充到查词面板和新词面板 → 查看释义 → 填写笔记 → 提交。

提交后，文章中所有该单词的位置都会同步变为橙色标记，后续其他文章中再次遇到该词也会被标出。

### 添加词组

在阅读视图中，拖动划选多个单词按钮即可选中词组。

> ⚠️ 在新词面板中添加时，**类别必须选「短语」**，否则不允许提交。

### 自动填充例句

点击单词后，插件会自动将其所在句子填入新词面板的例句栏。

若一个句子中有多个生词，只需在学习第一个生词时记录例句和翻译，后续生词会自动复用，**同一句话只需记录一次**。

### 结束阅读

读完文章并记完所有生词后，点击**「完成阅读」**按钮。

此操作会将文章中所有剩余淡蓝色单词以「无视」状态批量保存到数据库，下次阅读时这些词不再显示背景色。

> ⚠️ 点击前请确认已记录完所有需要学习的单词。

## 五、统计面板

点击左侧**飞机形按钮**，查看近 7 天学习情况：

|指标|说明|
|---|---|
|单日-无视|当天标记为「无视」的单词数|
|单日-学习|当天记过笔记的单词数（新学/眼熟/掌握/会了）|
|累计|从开始使用至今记录的全部单词数（含无视），大致反映词汇量|

> 注：因单词存在不同单复数、时态变形的重复，累计数字可能不完全准确。

## 六、联动 Various Complements（写作自动补全）

### 配置步骤

**1. 安装 Various Complements 插件**

**2. 在 Various Complements 设置中：**

- 启用 **Custom dictionary complement**
- 将 **Column delimiter** 改为 **Comma（逗号）**
- 在 **Custom dictionary paths** 中填写词库笔记路径，例如：`LingQ/words_database.md`

**3. 在 Language Learner 设置中：**

- 在 **Word Database Path** 中填写**同一个**笔记路径：`LingQ/words_database.md`

**4. 设置快捷键（推荐）：**

| 命令                                              | 推荐快捷键              | 作用        |
| ----------------------------------------------- | ------------------ | --------- |
| Language Learner: 刷新单词数据库                       | `Ctrl + W`         | 将生词写入词库笔记 |
| Various Complements: Reload custom dictionaries | `Ctrl + Shift + R` | 刷新补全缓存    |

### 日常使用

每次新记了单词后：

1. 按 `Ctrl + W` → 生词写入词库笔记
2. 按 `Ctrl + Shift + R` → 刷新 Various Complements 缓存
3. 之后写作时输入单词前几个字母，即可获得**英→中**和**中→英**双向补全提示

## 七、联动 Spaced Repetition（间隔复习）

### 配置步骤

**1. 安装 Spaced Repetition 插件**

**2. 创建复习数据库笔记**

新建一篇笔记（如 `LingQ/review_database.md`），在**第一行**写：

```
#flashcards
```

然后换行，其余内容留空。

**3. 在 Language Learner 设置中：**

在 **Review Database Path** 中填写该笔记路径：`LingQ/review_database.md`

（Spaced Repetition 插件本身无需额外配置）

**4. 设置快捷键（推荐）：**

|命令|推荐快捷键|作用|
|---|---|---|
|Language Learner: 刷新复习数据库| `Ctrl + R` |将新增生词导出为闪卡格式|

### 日常使用

1. 按 `Ctrl + R` 刷新复习数据库
    - 只会增量写入「上次刷新后新增」的「非无视」单词，不会破坏已有复习记录
2. 打开 Spaced Repetition 的**复习卡片面板**（左侧按钮）开始复习
3. 点击单词可播放发音（设置中可选英音或美音）
4. 按难易程度评级：**较难（-1 天）/ 记得（-2.3 天）/ 简单（-3.3 天）**

### 导出卡片格式示例

```
#flashcards
#word

president
?
n. 总统、主席
Sentences:
I appreciate your president having made me an honorary visiting professor...
《We choose to go to the moon》
```

## 八、数据库管理

### 自定义数据库

在插件设置的 **IndexDB 数据库** 区域：

- **数据库名称**：修改后重启 Obsidian 生效，不存在的名称会自动新建空库
- 适用场景：用多个 Obsidian 库分别学习不同材料时，保持数据库独立

### 导入 / 导出

|操作|格式|说明|
|---|---|---|
|导出| `.json` |备份当前数据库所有记录|
|导入| `.json` |⚠️ 会**覆盖**当前数据库，操作前请先导出备份|

|操作|快捷键（推荐）|
|---|---|

## 十、已知限制与注意事项

- 目前**仅支持中文母语者学习英语**，不支持其他语言组合
- 插件处于早期阶段，版本更新可能带来不兼容改动，更新前务必查看 Release Notes
- 需要手动安装，不在社区插件市场
- 词典依赖有道 API，需要联网
- 阅读页面不支持分页，较长文章体验一般
- tag 字段目前仅用于记录，按 tag 查询功能尚未实现
- 导入文本只支持纯文本手动粘贴，不支持电子书格式直接导入

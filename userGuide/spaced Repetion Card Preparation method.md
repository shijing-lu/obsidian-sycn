## 一、基础卡片语法
### 1. 单行双向卡（最常用）
使用 `::` 分隔问题和答案，正反两面都会被复习。

```
什么是光合作用? :: 植物利用光能将CO₂和水转化为糖和氧气
牛顿第二定律? :: F = ma，力等于质量乘以加速度
```

### 2. 单行单向卡

使用 `:::` 分隔，只复习正面（问题→答案），适合定义类记忆。

```
牛顿第一定律 ::: 物体在不受外力时保持静止或匀速直线运动
```

### 3. 多行双向卡

问题和答案各占多行，用单独一行的 `?` 隔开，末尾空一行结束。

```
什么是贝叶斯定理？
它有哪些主要应用？
?
P(A|B) = P(B|A)×P(A) / P(B)
应用：垃圾邮件过滤、医学诊断、风险评估
```

### 4. 多行单向卡

使用 `??` 作为分隔符，只复习正方向。

```
解释TCP三次握手的过程
??
1. 客户端发送 SYN
2. 服务端回复 SYN-ACK
3. 客户端发送 ACK
```

## 二、填空卡（Cloze）

用 `==` 包裹需要挖空的内容（即 Markdown 高亮语法），每个挖空生成一张独立卡片。

> 需在插件设置中开启：**Convert highlights to clozes**

```
DNA的双螺旋结构由 ==碱基对== 通过氢键连接。

水的化学式是 ==H₂O==，沸点是 ==100°C==。

Python 列表推导式：[==expression== for item in iterable if ==condition==]
```

同一句话有多个挖空时，每个挖空单独考察，其余内容作为提示保留。

## 三、整篇笔记复习（Note Review）

在 frontmatter 中打上复习标签，整篇笔记作为复习对象，无需写卡片格式。

```yaml
---
tags:
  - review
sr-due: 2024-04-10
sr-interval: 7
sr-ease: 250
---

# 操作系统调度算法
FCFS、SJF、优先级调度、时间片轮转……
```

插件会自动维护 `sr-due`、`sr-interval`、`sr-ease` 三个字段，无需手动修改。

## 四、牌组管理

### 按标签分组

通过不同的 tag 前缀划分牌组，支持多级嵌套（最多三级）。

```
#flashcards/数学/微积分
什么是导数? :: 函数在某点的瞬时变化率

#flashcards/语言/日语
こんにちは :: 你好（白天问候）

#flashcards/编程/算法
快速排序的平均时间复杂度? :: O(n log n)
```

### 按文件夹自动归组

在插件设置中开启 **Flashcard deck path**，文件夹名称即牌组名称。

### 忽略特定卡片

在行末添加 HTML 注释跳过该卡片：

```
这张暂时不复习 <!-- SR-IGNORE -->
问题? :: 答案
```

## 五、卡片类型速查表

|格式|分隔符|方向|适用场景|
|---|---|---|---|
|单行双向| `::` |正反两面都考|概念互译、词汇|
|单行单向| `:::` |只考正面|定义、公式记忆|
|多行双向| `?`（单独一行）|正反两面都考|复杂知识点|
|多行单向| `??`（单独一行）|只考正面|步骤、流程|
|填空 Cloze| `==内容==` |每个挖空单独一张|段落记忆、关键词|
|整篇笔记|frontmatter tag|整篇阅读|综合性笔记复习|

## 六、复习评级（SM-2 算法）

复习时对每张卡片打分，插件自动计算下次复习日期：

|评级|说明|
|---|---|
|**Easy（容易）**|间隔大幅增加，ease 上升|
|**Good（一般）**|正常增加间隔|
|**Hard（困难）**|间隔缩短，ease 下降|

**复习快捷键：**

- `空格` 翻牌
- `1` → Hard，`2` → Good，`3` → Easy

## 七、重要配置项

|配置项|默认值|说明|
|---|---|---|
|Flashcard tags| `#flashcards` |触发卡片复习的标签|
|Note review tags| `#review` |触发整篇笔记复习的标签|
|Inline separator| `::` |单行卡片分隔符|
|Multi-line separator| `?` |多行卡片分隔符|
|Initial ease| `250` |初始难度系数|
|Easy bonus| `1.3` |标记为 Easy 时的间隔倍数|
|Max interval| `36500` |最大复习间隔（天）|

## 八、实用技巧

- 卡片可以和普通笔记内容混写，插件只识别含有对应标签或分隔符的内容
- 同一个文件里可以混用多种卡片格式
- 推荐用 `#flashcards/科目/章节` 三级结构组织备考体系
- frontmatter 中的 `sr-*` 字段由插件自动维护，不要手动修改
- 填空卡适合配合原文段落使用，保留上下文语境有助于记忆
